## 1.6 Keywords

### 1.6.1 Definition

A keyword is a string of ASCII characters that is reserved by the Hopper language and MUST NOT be used as a user-defined identifier. Keywords are matched by the ANTLR4 implicit literal mechanism: every quoted string literal appearing in the parser grammar generates a lexer rule that takes precedence over the `Identifier` rule for that exact string.

A conforming implementation MUST reserve all keywords listed in this section regardless of whether the surrounding grammar context would resolve the ambiguity. Reserved means: the string cannot be used as an identifier anywhere in the source file, not merely in positions where the parser could accept an identifier.

The three exceptions are the **contextual keywords** `value`, `address`, and `size`, which remain valid identifier tokens in certain positions. See section 1.5.5.

### 1.6.2 Full Keyword List

The following table lists every reserved keyword in Hopper, grouped by category. Each keyword corresponds to one or more parser productions.

#### Type Keywords

| Keyword | Meaning |
|---------|---------|
| `int` | 64-bit signed integer type (i64) |
| `bool` | Boolean type (i1) |
| `float` | Double-precision floating-point type (double) |
| `byte` | 8-bit signed integer type (i8) |
| `char` | 8-bit signed integer type; alias for byte in character contexts (i8) |
| `bit` | Single-bit type; used in bitfield declarations |
| `string` | Null-terminated byte string type (i8*) |
| `String` | Reserved name for the String template class |
| `address` | Raw pointer type (i8*); also a contextual keyword in field/param position |
| `void` | Absent return type; used only as an explicit return-type annotation |
| `unsigned` | Modifier keyword; must be followed by `int` or `byte` |
| `callback` | Function-pointer type keyword; `callback(T...) R` |

#### Declaration Keywords

| Keyword | Meaning |
|---------|---------|
| `function` | Declares a function or method |
| `extern` | Declares an external (C ABI) function |
| `template` | Declares a parameterized class or function |
| `class` | Declares a class |
| `struct` | Declares a plain-data structure (no methods, no default values) |
| `contract` | Declares an contract contract |
| `enum` | Declares an enumeration |
| `alias` | Declares a type alias |
| `entry` | Declares the program entry point |
| `bind` | Declares a hardware address binding for a function pointer |
| `strict` | Declares a named alias for a memory-mapped hardware register |
| `bitfield` | Declares a bit-level packed structure |
| `operator` | Declares an operator overload on a class |
| `constructor` | Declares the constructor method of a class |
| `destructor` | Declares the destructor method of a class |
| `satisfies` | Specifies contract conformance in a class declaration |
| `const` | Declares an immutable local variable inside a block (`ConstVarDecl` statement) |
| `pad` | Inserts unnamed padding bytes (in `struct`) or padding bits (in `bitfield`) |

#### Import Keywords

| Keyword | Meaning |
|---------|---------|
| `import` | Begins an import declaration |
| `from` | Separates the imported name from the module name in an import declaration |

#### Control Flow Keywords

| Keyword | Meaning |
|---------|---------|
| `return` | Returns a value (or nothing) from a function |
| `if` | Conditional branch |
| `else` | Alternative branch of an `if` statement |
| `while` | Conditional loop |
| `for` | Counted loop |
| `break` | Exits the innermost loop |
| `continue` | Advances to the next iteration of the innermost loop |
| `defer` | Defers execution of an expression until the enclosing scope exits |

#### Memory Keywords

| Keyword | Meaning |
|---------|---------|
| `allocate` | Allocates heap memory; returns an `address` |
| `deallocate` | Frees heap memory previously allocated with `allocate` |

#### Inline Assembly

| Keyword | Meaning |
|---------|---------|
| `asm` | Begins an inline assembly block (consumed as `AsmBlock` token) |

#### Receiver Reference

| Identifier | Meaning |
|------------|---------|
| `self` | The receiver object within a class method or constructor |

> **Implementation note.** `self` is NOT a reserved keyword in the grammar (`Hopper.g4` contains no `'self'` literal rule). The compiler treats `self` as a plain `Identifier` and pre-loads it as a variable pointing to the class instance pointer at the entry of every class method and constructor. Programs can technically use `self` as a variable or parameter name outside a class context without a grammar-level error, though doing so is strongly discouraged.

#### Boolean and Null Literals

| Keyword | Meaning |
|---------|---------|
| `true` | Boolean true literal |
| `false` | Boolean false literal |
| `null` | Null pointer literal |

#### Contextual Keywords (not fully reserved)

| Keyword | Meaning |
|---------|---------|
| `value` | Dereference operator in `::value` context; MAY also appear as field or parameter name |
| `address` | Raw pointer type and `::address` operator; MAY also appear as field or parameter name |
| `size` | Size-of operator in `::size` context; MAY also appear as field or parameter name |

#### Contract and Constraint Keywords

| Keyword | Meaning |
|---------|---------|
| `constrain` | Introduces a range constraint clause on a variable declaration |
| `invariant` | Introduces a loop invariant clause |
| `requires` | Introduces a precondition clause on a function |
| `ensures` | Introduces a postcondition clause on a function |

### 1.6.3 Complete Alphabetical Reference

For quick lookup, the complete list of keywords in alphabetical order:

```
address    alias      allocate   asm        bind
bit        bitfield   bool       break      callback
char       class      const      constrain  constructor
continue   deallocate defer      destructor else
ensures    entry      enum       extern     false
float      for        from       function   if
satisfies import     int        contract  invariant
null       operator   pad        requires   return     self
size       strict     string     String     struct
template   true       unsigned   value      void
while
```

A conforming implementation MUST reject any program that uses any of these strings as a user-defined identifier in a position where an `Identifier` token is expected, except for the three contextual keywords (`value`, `address`, `size`) in the positions explicitly defined by the `fieldName` and `paramName` grammar productions.

> **Implementation note.** `self` appears in the list above but is NOT a grammar-level reserved keyword. The grammar has no `'self'` literal rule; `self` is matched as a plain `Identifier` token. The restriction on using `self` as a user-defined name is enforced by convention and compiler-internal variable shadowing, not by the grammar.

### 1.6.4 Future Reserved Words

The following identifiers are not currently keywords but are reserved for potential future use. A conforming implementation SHOULD warn when these identifiers appear as user-defined names:

- `async`, `await` — potential future concurrency keywords
- `match`, `case` — potential future pattern-matching keywords
- `yield` — potential future coroutine keyword
- `macro` — potential future metaprogramming keyword
- `where` — potential future constraint keyword
- `type` — potential future type declaration keyword
- `use` — potential future path-import keyword

Programs SHOULD NOT use these identifiers in public interfaces to ensure forward compatibility.

### 1.6.5 The `String` Keyword

`String` (capital S) is a keyword that names the built-in generic string template class. It is handled specially in the grammar by the `templateName` and `className` productions, which explicitly allow `String` where a class or template name is expected. Despite being a keyword, `String` behaves like a type name in all type positions. Programs MUST NOT use `String` as the name of any user-defined entity other than a class named exactly `String` that satisfies the `String` template (which is only permitted in the standard library).
