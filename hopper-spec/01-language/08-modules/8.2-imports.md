## 8.2 Imports

An import declaration brings named exports from another module into the current compilation unit.

### 8.2.1 Import Syntax

```
import Identifier (',' Identifier)* from Identifier
```

Examples:

```hopper
import IO from linux
import IO, FileSystem from linux
import String from ds
import abs from math
import SIMD from x86_64
```

Import declarations MUST appear on their own lines. They MAY appear anywhere among the top-level declarations of a module file. Multiple import declarations MAY appear in a single file. Order does not affect visibility — the compiler resolves all imports before codegen begins.

### 8.2.2 What a Module Is

A module is a **directory**, not a file. `from linux` means the `linux` module directory. The module name is a replacement for what would otherwise be a filename — modules exist because a logical unit of code now spans multiple files.

The import name (`IO`, `abs`, `String`) identifies which export to bring in from that module. The module directory may contain any number of source files; the import name selects one named export from among them.

### 8.2.3 Resolution Algorithm

Given `import Foo from bar`:

1. Locate the `bar` module: walk up from the importing file's directory looking for `modules/bar/`.
2. Look up `Foo` in the nearest `hopper.json` targets section (`targets.<target>.Foo`).
   - **If a binding exists** → interface resolution (§8.2.4).
   - **If no binding exists** → direct resolution (§8.2.5).
3. The compiler maintains a *visited set* of absolute file paths. Any file already in the visited set is skipped to prevent duplicate definitions.

### 8.2.4 Interface Resolution

When a targets binding exists for `Foo`:

```json
"Foo": {
  "from":           "bar",
  "contract":      "modules/bar/contracts/Foo.hop",
  "implementation": "modules/baz/src/FooImpl.hop"
}
```

1. Verify the `from` field matches the module name in the import. A mismatch is a compile error.
2. Resolve both `contract` and `implementation` to absolute paths. A missing file is a compile error.
3. Parse and merge both files into the current compilation unit (subject to the visited set).
4. All free functions, classes, interfaces, enums, and type aliases from both files come into scope.

This is how hardware abstraction works: the source file imports `IO from linux`, and the build file decides which implementation backs it.

**Multiple names from one module:**

```hopper
import IO, FileSystem from linux
```

Each name is resolved independently. If both bind to the same implementation file, that file is loaded only once (enforced by the visited set).

### 8.2.5 Direct Resolution

When no targets binding exists for `Foo`:

1. Scan the module's source files for a declaration named `Foo`.
2. By convention, source files are named after their primary export (`abs.hop` exports `abs`), so the compiler first tries `modules/bar/src/Foo.hop`.
3. If not found by name, scan all `.hop` files in `modules/bar/src/` for a matching declaration.
4. Parse and merge the resolved file into the current compilation unit.

Use this path for concrete types, functions, and enums that have no platform-specific implementation — they are what they are regardless of target.

### 8.2.6 Effect on Scope

After any import, the following from the resolved file(s) are in scope:

- Free functions (`function` and `extern function`)
- Class declarations
- Interface declarations
- Enum declarations
- Type aliases
- Template declarations and template functions

The `entry` declaration from an imported file is never inherited.

### 8.2.7 Error Conditions

A conforming implementation MUST report a compile error when:

- The `from` field in a targets binding does not match the module name in the import statement.
- A file path in a targets binding does not exist.
- No module directory `modules/<name>/` is found anywhere in the directory tree.
- No export named `Foo` is found in the module's source files.
- An import is syntactically malformed.

The error message SHOULD include the name of the importing source file and the name of the unresolved import.
