## Networking

Hopper exposes networking through two interfaces:

- **`Socket`** (`modules/linux/interfaces/Socket.hop`) — socket creation, connection, and
  data transfer.
- **`Network`** (`modules/linux/interfaces/Network.hop`) — I/O multiplexing: `select`,
  `poll`, and `epoll`.

Both are satisfied on x86-64 Linux by `modules/x86_64/src/LinuxSyscalls.hop`.

---

```json
{

```

### Socket Enums

#### `AddrFamily`

| Constant | Value | Meaning |
|----------|-------|---------|
| `AddrFamily.AF_UNIX` | `1` | Unix domain sockets (local IPC) |
| `AddrFamily.AF_INET` | `2` | IPv4 |
| `AddrFamily.AF_INET6` | `10` | IPv6 |

#### `SockType`

| Constant | Value | Meaning |
|----------|-------|---------|
| `SockType.SOCK_STREAM` | `1` | TCP (sequenced, reliable, two-way) |
| `SockType.SOCK_DGRAM` | `2` | UDP (connectionless, unreliable) |
| `SockType.SOCK_RAW` | `3` | Raw socket (requires `CAP_NET_RAW`) |
| `SockType.SOCK_NONBLOCK` | `2048` | Set non-blocking mode at creation |
| `SockType.SOCK_CLOEXEC` | `524288` | Set close-on-exec at creation |

`SockType` values may be OR'd together: `SockType.SOCK_STREAM | SockType.SOCK_CLOEXEC`.

#### `SockLevel`

Protocol level for `setsockopt` / `getsockopt`:

| Constant | Value | Meaning |
|----------|-------|---------|
| `SockLevel.SOL_SOCKET` | `1` | Socket-level options |
| `SockLevel.IPPROTO_TCP` | `6` | TCP-level options |
| `SockLevel.IPPROTO_UDP` | `17` | UDP-level options |

#### `SockOpt`

Socket option names:

| Constant | Value | Level | Meaning |
|----------|-------|-------|---------|
| `SockOpt.SO_REUSEADDR` | `2` | `SOL_SOCKET` | Allow reuse of local address |
| `SockOpt.SO_KEEPALIVE` | `9` | `SOL_SOCKET` | Enable TCP keepalive probes |
| `SockOpt.SO_SNDBUF` | `7` | `SOL_SOCKET` | Send buffer size |
| `SockOpt.SO_RCVBUF` | `8` | `SOL_SOCKET` | Receive buffer size |
| `SockOpt.TCP_NODELAY` | `1` | `IPPROTO_TCP` | Disable Nagle's algorithm |
| `SockOpt.TCP_KEEPIDLE` | `4` | `IPPROTO_TCP` | Seconds before first keepalive probe |
| `SockOpt.TCP_KEEPINTVL` | `5` | `IPPROTO_TCP` | Interval between keepalive probes |

#### `ShutHow`

How argument for `shutdown`:

| Constant | Value | Meaning |
|----------|-------|---------|
| `ShutHow.SHUT_RD` | `0` | Disallow further receives |
| `ShutHow.SHUT_WR` | `1` | Disallow further sends (sends FIN) |
| `ShutHow.SHUT_RDWR` | `2` | Disallow both |

#### `MsgFlag`

Flags for `send`, `recv`, and variants:

| Constant | Value | Meaning |
|----------|-------|---------|
| `MsgFlag.MSG_OOB` | `1` | Send/receive out-of-band data |
| `MsgFlag.MSG_PEEK` | `2` | Peek at data without consuming it |
| `MsgFlag.MSG_DONTWAIT` | `64` | Non-blocking operation |
| `MsgFlag.MSG_WAITALL` | `256` | Block until full request satisfied |
| `MsgFlag.MSG_NOSIGNAL` | `16384` | Do not generate `SIGPIPE` on broken connection |

### Socket Function Reference

#### `socket`

```hopper
function socket(int domain, int type, int protocol) int
```

Create a new socket. `domain` is an `AddrFamily` value. `type` is a `SockType` value
(optionally OR'd with `SOCK_NONBLOCK` or `SOCK_CLOEXEC`). `protocol` is `0` to let the
kernel choose, or an explicit protocol number.

Returns a non-negative file descriptor or a negative errno.

```hopper
int fd = socket(AddrFamily.AF_INET, SockType.SOCK_STREAM | SockType.SOCK_CLOEXEC, 0)
```

#### `socketBind`

```hopper
function socketBind(int fd, address addr, int addrlen) int
```

Bind `fd` to the local address at `addr`. `addrlen` is the byte size of the sockaddr structure.

> **Note:** This function is named `socketBind` rather than `bind` because `bind` is a
> reserved keyword in the Hopper grammar (used for hardware address linker directives).
> All interface declarations and callers MUST use `socketBind`.

For IPv4, `addr` points to `struct sockaddr_in` (16 bytes):

| Offset | Type | Field | Value |
|--------|------|-------|-------|
| `0` | `i16` | `sin_family` | `2` (AF_INET) |
| `2` | `u16 BE` | `sin_port` | Port in network byte order |
| `4` | `u32 BE` | `sin_addr` | IPv4 address in network byte order (`0` = `INADDR_ANY`) |
| `8` | `i8[8]` | `sin_zero` | Padding (zero) |

Returns `0` or a negative errno.

#### `listen`

```hopper
function listen(int fd, int backlog) int
```

Mark `fd` as a passive socket that will accept connections. `backlog` is the maximum length
of the pending connection queue. Returns `0` or a negative errno.

#### `accept`

```hopper
function accept(int fd, address addr, address addrlen) int
```

Accept an incoming connection on `fd`. `addr` receives the peer's address; `addrlen` is a
pointer to an `int` holding the size of `addr` on input and the actual size on output.
Pass `0` for both `addr` and `addrlen` to discard the peer address.

Returns a new file descriptor for the accepted connection, or a negative errno.

#### `accept4`

```hopper
function accept4(int fd, address addr, address addrlen, int flags) int
```

Like `accept` but accepts `flags`. Supported: `SockType.SOCK_NONBLOCK` and
`SockType.SOCK_CLOEXEC`.

#### `connect`

```hopper
function connect(int fd, address addr, int addrlen) int
```

Connect `fd` to the remote address at `addr`. For `SOCK_STREAM` sockets this initiates the
TCP handshake. For `SOCK_DGRAM` sockets it sets the default destination address.

Returns `0` on success, `-EINPROGRESS` if the socket is non-blocking and the connection has
not yet completed, or another negative errno on error.

#### `shutdown`

```hopper
function shutdown(int fd, int how) int
```

Partially or fully shut down a socket connection. `how` is a `ShutHow` value. Returns `0` or
a negative errno.

#### `getsockname`

```hopper
function getsockname(int fd, address addr, address addrlen) int
```

Retrieve the local address bound to `fd`. `addrlen` is an in/out parameter.

#### `getpeername`

```hopper
function getpeername(int fd, address addr, address addrlen) int
```

Retrieve the remote address of the peer connected to `fd`.

#### `socketpair`

```hopper
function socketpair(int domain, int type, int protocol, address fds) int
```

Create a pair of connected sockets and write the two file descriptors into `fds` (a 2-element
`int` array). Commonly used with `AF_UNIX` for bidirectional IPC between related processes.

### Data Transfer

#### `send`

```hopper
function send(int fd, address buf, int len, int flags) int
```

Send `len` bytes from `buf` through connected socket `fd`. `flags` is a bitmask of `MsgFlag`
values, or `0`. Returns bytes sent or a negative errno.

#### `recv`

```hopper
function recv(int fd, address buf, int len, int flags) int
```

Receive up to `len` bytes from connected socket `fd` into `buf`. Returns bytes received, `0`
on peer close, or a negative errno.

#### `sendto`

```hopper
function sendto(int fd, address buf, int len, int flags, address addr, int addrlen) int
```

Send `len` bytes to `addr` (for connectionless sockets). For connected sockets, `addr` and
`addrlen` may be `0`.

#### `recvfrom`

```hopper
function recvfrom(int fd, address buf, int len, int flags, address addr, address addrlen) int
```

Receive up to `len` bytes, storing the sender's address in `addr`. `addrlen` is an in/out
parameter.

#### `sendmsg`

```hopper
function sendmsg(int fd, address msg, int flags) int
```

Send a message described by a `msghdr` structure at `msg`. Supports scatter-gather I/O and
ancillary data (e.g., passing file descriptors over Unix sockets).

#### `recvmsg`

```hopper
function recvmsg(int fd, address msg, int flags) int
```

Receive into a `msghdr` structure. Supports scatter-gather I/O and ancillary data.

#### `sendmmsg`

```hopper
function sendmmsg(int fd, address msgs, int vlen, int flags) int
```

Send multiple messages with a single syscall. `msgs` points to an array of `mmsghdr`
structures. Returns the number of messages sent or a negative errno.

#### `recvmmsg`

```hopper
function recvmmsg(int fd, address msgs, int vlen, int flags, address timeout) int
```

Receive multiple messages in a single syscall. `timeout` is a pointer to `struct timespec`
or `0` for blocking.

### Socket Options

#### `setsockopt`

```hopper
function setsockopt(int fd, int level, int optname, address optval, int optlen) int
```

Set a socket option. `level` is a `SockLevel` value; `optname` is a `SockOpt` value.
`optval` points to the option value (often an `int`); `optlen` is its size.

```hopper
// Enable SO_REUSEADDR
int one = 1
setsockopt(fd, SockLevel.SOL_SOCKET, SockOpt.SO_REUSEADDR, cast<address>(one), 4)
```

#### `getsockopt`

```hopper
function getsockopt(int fd, int level, int optname, address optval, address optlen) int
```

Retrieve a socket option. `optlen` is an in/out parameter.

### Minimal TCP Server Pattern

```hopper
import Socket from linux
import IO from linux

entry main {
    int fd = socket(AddrFamily.AF_INET, SockType.SOCK_STREAM | SockType.SOCK_CLOEXEC, 0)

    // SO_REUSEADDR
    int one = 1
    setsockopt(fd, SockLevel.SOL_SOCKET, SockOpt.SO_REUSEADDR, cast<address>(one), 4)

    // Build sockaddr_in for 0.0.0.0:8080
    address sa = allocate 16
    // sin_family = AF_INET = 2
    // sin_port   = htons(8080) = 0x901F in network order
    // sin_addr   = 0 (INADDR_ANY)
    socketBind(fd, sa, 16)
    listen(fd, 128)

    int client = accept(fd, 0, 0)
    // ... read/write on client ...
    close(client)
    close(fd)
}
```

### Network Enums

#### `EpollFlag`

| Constant | Value | Meaning |
|----------|-------|---------|
| `EpollFlag.EPOLL_CLOEXEC` | `524288` | Set close-on-exec on the epoll fd |

#### `EpollOp`

| Constant | Value | Meaning |
|----------|-------|---------|
| `EpollOp.EPOLL_CTL_ADD` | `1` | Register `fd` with the epoll instance |
| `EpollOp.EPOLL_CTL_DEL` | `2` | Remove `fd` from the epoll instance |
| `EpollOp.EPOLL_CTL_MOD` | `3` | Modify the events registered for `fd` |

#### `EpollEvent`

Bitmask values for the `events` field of `struct epoll_event`:

| Constant | Value | Meaning |
|----------|-------|---------|
| `EpollEvent.EPOLLIN` | `1` | Data available for reading |
| `EpollEvent.EPOLLPRI` | `2` | Exceptional condition |
| `EpollEvent.EPOLLOUT` | `4` | Socket ready for writing |
| `EpollEvent.EPOLLERR` | `8` | Error (always monitored) |
| `EpollEvent.EPOLLHUP` | `16` | Hangup (always monitored) |
| `EpollEvent.EPOLLRDHUP` | `8192` | Peer closed write end of connection |
| `EpollEvent.EPOLLET` | `2147483648` | Edge-triggered mode |
| `EpollEvent.EPOLLONESHOT` | `1073741824` | One-shot notification; rearm after each event |

### Network (Multiplexing) Function Reference

#### `select`

```hopper
function select(int nfds, address readfds, address writefds, address exceptfds, address timeout) int
```

Wait for one or more file descriptors to become ready. `nfds` is one more than the highest fd
in any set. Each set is a bitmask of file descriptors (`fd_set`, 128 bytes). `timeout` is a
pointer to `struct timeval` or `0` to block indefinitely.

Returns the number of ready file descriptors, `0` on timeout, or a negative errno.

`select` does not scale well past a few hundred file descriptors. Prefer `epoll` for servers
with many connections.

#### `pselect6`

```hopper
function pselect6(int nfds, address readfds, address writefds, address exceptfds,
                  address timeout, address sigmask) int
```

Like `select` but uses `struct timespec` for `timeout` and atomically applies `sigmask`
during the wait, allowing safe handling of signals.

#### `poll`

```hopper
function poll(address fds, int nfds, int timeout) int
```

Wait for events on `nfds` file descriptors described by the `pollfd` array at `fds`.
`timeout` is in milliseconds; `-1` blocks indefinitely; `0` returns immediately.

Each `pollfd` structure (8 bytes on x86-64):

| Offset | Type | Field | Meaning |
|--------|------|-------|---------|
| `0` | `i32` | `fd` | File descriptor |
| `4` | `i16` | `events` | Requested events (bitmask) |
| `6` | `i16` | `revents` | Returned events (written by kernel) |

Common event bits: `POLLIN=1`, `POLLOUT=4`, `POLLERR=8`, `POLLHUP=16`.

Returns the number of descriptors with events, `0` on timeout, or a negative errno.

#### `ppoll`

```hopper
function ppoll(address fds, int nfds, address tmo, address sigmask, int sigsetsize) int
```

Like `poll` but uses `struct timespec` for the timeout and atomically applies `sigmask`.

#### `epoll_create1`

```hopper
function epoll_create1(int flags) int
```

Create a new epoll instance. `flags`: `0` or `EpollFlag.EPOLL_CLOEXEC`. Returns an epoll
file descriptor or a negative errno.

#### `epoll_ctl`

```hopper
function epoll_ctl(int epfd, int op, int fd, address event) int
```

Control the set of file descriptors watched by epoll instance `epfd`. `op` is an `EpollOp`
value. `fd` is the file descriptor to add, modify, or remove. `event` points to a
`struct epoll_event` (12 bytes):

| Offset | Type | Field | Meaning |
|--------|------|-------|---------|
| `0` | `i32` | `events` | Bitmask of `EpollEvent` values |
| `4` | `i64` | `data.u64` | User data returned with the event (often the fd) |

Pass `0` for `event` when calling with `EPOLL_CTL_DEL`.

Returns `0` or a negative errno.

#### `epoll_wait`

```hopper
function epoll_wait(int epfd, address events, int maxevents, int timeout) int
```

Wait for events on the epoll instance `epfd`. Fills `events` with up to `maxevents` ready
events. `timeout` is in milliseconds; `-1` blocks indefinitely; `0` returns immediately.

Returns the number of events filled into `events`, `0` on timeout, or a negative errno.

#### `epoll_pwait`

```hopper
function epoll_pwait(int epfd, address events, int maxevents, int timeout,
                     address sigmask, int sigsetsize) int
```

Like `epoll_wait` but atomically applies `sigmask` for the duration of the wait.

### Minimal epoll Event Loop

```hopper
import Socket from linux
import Network from linux

entry main {
    int epfd = epoll_create1(EpollFlag.EPOLL_CLOEXEC)

    // Add a listening socket to epoll
    address ev = allocate 12
    // ev.events = EPOLLIN | EPOLLET
    // ev.data.u64 = listenfd
    epoll_ctl(epfd, EpollOp.EPOLL_CTL_ADD, listenfd, ev)

    address events = allocate 1200   // room for 100 epoll_event structs
    while (true) {
        int n = epoll_wait(epfd, events, 100, -1)
        int i = 0
        while (i < n) {
            // process events[i]
            i = i + 1
        }
    }
}
```
