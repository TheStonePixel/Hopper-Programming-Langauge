## Concurrency

Hopper has no language-level concurrency primitives such as goroutines, async/await, or
green threads. Concurrency is achieved by composing the OS-level thread and synchronization
interfaces directly. This section describes the facilities available.

See also §7 (Language Concurrency) for the language-level memory model and the constraints
that apply to shared-state access.

---

### Overview

Hopper concurrency uses Linux threads — processes that share address space, file descriptors,
and signal handlers via the `clone` syscall. Synchronization is built on the Linux `futex`
primitive. These mechanisms are exposed through two modules:

| Module | Location | Role |
|--------|----------|------|
| `Process` contract | `linux/interfaces/Process.hop` | `clone`, `fork`, `execve`, signals |
| `x86_64/src/thread.hop` | hardware layer | `futex`, `futex_wait`, `futex_wake`, `gettid`, `sched_yield` |
| `x86_64/src/sem.hop` | hardware layer | SysV semaphores (`semget`, `semop`, `semctl`) |


The threading helpers (`futex`, `gettid`, `sched_yield`) from `x86_64/src/thread.hop` are
currently accessed by including that file directly in the build target's implementation list.
A first-class threading contract is planned but not yet standardized.

### Process Creation and Thread Spawning

#### `fork`

```hopper
function fork() int
```

Create a copy of the calling process. The child process has a new PID and inherits a copy of
the parent's memory. Returns the child PID in the parent and `0` in the child. Returns a
negative errno on error.

`fork` is the POSIX way to create new processes. For threads (shared address space) use
`clone` instead.

#### `vfork`

```hopper
function vfork() int
```

Like `fork` but the child and parent share the same virtual address space until the child
calls `execve` or `exit`. The parent is suspended until then. Use only when immediately
followed by `execve`.

#### `clone`

```hopper
function clone(int flags, address stack, address ptid, address ctid, int regs) int
```

Create a new thread or process. `flags` controls what is shared between parent and child.
`stack` is the top of the new thread's stack (required for CLONE_VM — the child's stack MUST
be pre-allocated). Returns the child TID (in the parent) or a negative errno.

The `CloneFlag` enum defines the sharing flags:

| Constant | Value | Effect |
|----------|-------|--------|
| `CloneFlag.CLONE_VM` | `256` | Share virtual address space (makes it a thread) |
| `CloneFlag.CLONE_FS` | `512` | Share filesystem root, cwd, and umask |
| `CloneFlag.CLONE_FILES` | `1024` | Share the file descriptor table |
| `CloneFlag.CLONE_SIGHAND` | `2048` | Share signal handlers |
| `CloneFlag.CLONE_THREAD` | `65536` | Place in the same thread group (POSIX thread semantics) |

A POSIX-compatible thread uses all five flags together:

```hopper
int flags = CloneFlag.CLONE_VM | CloneFlag.CLONE_FS | CloneFlag.CLONE_FILES |
            CloneFlag.CLONE_SIGHAND | CloneFlag.CLONE_THREAD

address stack = allocate 65536        // 64 KB stack
address stackTop = stack + 65536      // stack grows down on x86-64

int tid = clone(flags, stackTop, 0, 0, 0)
```

The new thread begins execution at the next instruction after `clone` in the child's control
flow (same as `fork`). Programs MUST ensure the child's function pointer and arguments are in
shared memory before calling `clone`.

#### `clone3`

```hopper
function clone3(address args, int argsz) int
```

Extended clone contract. `args` points to a `clone_args` structure that carries all
parameters including the new thread's stack, thread-local storage pointer, and exit signal.
Recommended for new code when fine-grained control is required.

### Thread Identity

#### `gettid` (from `x86_64/src/thread.hop`)

```hopper
function gettid() int
```

Return the calling thread's thread ID. Each `clone`d thread has its own TID. `getpid()`
returns the thread group ID (TGID), which is the same for all threads in a process.

#### `set_tid_address`

```hopper
function set_tid_address(address tidptr) int
```

Register an address that the kernel will clear (write zero) when the thread exits. Threading
libraries use this to implement join and thread-death detection via `futex_wait`.

### Scheduling

#### `sched_yield` (from `x86_64/src/thread.hop`)

```hopper
function sched_yield() int
```

Voluntarily yield the CPU. The calling thread is moved to the end of the run queue for its
priority. Useful in spin loops to avoid starving other threads.

#### `sched_getscheduler` / `sched_setscheduler`

```hopper
function sched_getscheduler(int pid) int
function sched_setscheduler(int pid, int policy, address param) int
```

Get or set the scheduling policy for thread `pid`. Pass `0` for the calling thread.

The `SchedPolicy` enum (from `System.hop`) defines:

| Constant | Value | Meaning |
|----------|-------|---------|
| `SchedPolicy.SCHED_OTHER` | `0` | Default time-sharing (CFS) |
| `SchedPolicy.SCHED_FIFO` | `1` | Real-time FIFO (requires `CAP_SYS_NICE`) |
| `SchedPolicy.SCHED_RR` | `2` | Real-time round-robin (requires `CAP_SYS_NICE`) |

#### `sched_getaffinity` / `sched_setaffinity`

```hopper
function sched_getaffinity(int pid, int cpusetsize, address mask) int
function sched_setaffinity(int pid, int cpusetsize, address mask) int
```

Get or set the CPU affinity mask for thread `pid`. `mask` is a bitmask of CPU indices;
`cpusetsize` is the byte size of `mask`.

### Futex-Based Synchronization

The `futex` syscall is the kernel primitive underlying all modern Linux synchronization.
The wrappers in `x86_64/src/thread.hop` expose the most common operations.

#### `futex`

```hopper
function futex(address uaddr, int futex_op, int val, address timeout,
               address uaddr2, int val3) int
```

Raw futex syscall. `uaddr` is the address of a shared 32-bit integer. `futex_op` selects the
operation. `timeout` points to `struct timespec` or is `0` for indefinite blocking.

Common `futex_op` values:

| Value | Constant | Meaning |
|-------|----------|---------|
| `0` | `FUTEX_WAIT` | Block if `*uaddr == val` |
| `1` | `FUTEX_WAKE` | Wake up to `val` waiters |
| `128` | `FUTEX_WAIT_PRIVATE` | Private (process-local) wait |
| `129` | `FUTEX_WAKE_PRIVATE` | Private (process-local) wake |

The `_PRIVATE` variants are faster for threads within a single process because the kernel can
skip cross-process hash bucket locking.

#### `futex_wait`

```hopper
function futex_wait(address uaddr, int val) int
```

Block the calling thread until `*uaddr` is no longer `val` (or a spurious wakeup occurs).
Uses `FUTEX_WAIT_PRIVATE`. Returns `0` on wakeup or a negative errno (`-EAGAIN` if
`*uaddr != val` at entry — not an error, just a missed opportunity).

#### `futex_wake`

```hopper
function futex_wake(address uaddr, int count) int
```

Wake up to `count` threads waiting on `uaddr`. Pass `1` to wake one waiter, or a large number
to wake all. Uses `FUTEX_WAKE_PRIVATE`. Returns the number of threads woken.

#### Building a Mutex with futex

A simple non-recursive mutex:

```hopper
// State: 0 = unlocked, 1 = locked
// mutex is an address pointing to an int in shared memory

function mutexLock(address mutex) {
    // CAS loop — must be implemented with inline asm (cmpxchg)
    // futex_wait blocks if mutex is still 1 after failed CAS
}

function mutexUnlock(address mutex) {
    // store 0 (release), then futex_wake to notify one waiter
    mutex::value = 0
    futex_wake(mutex, 1)
}
```

Full mutex implementation requires a compare-and-swap (`cmpxchg`) via inline assembly. The
`x86_64/` module does not yet provide a pre-built mutex class — programs that need one MUST
implement it in Hopper using `asm {}` blocks within a module file.

### SysV Semaphores (from `x86_64/src/sem.hop`)

SysV semaphores are kernel-managed counting semaphores that can span multiple processes.
For new code within a single process, prefer futex-based synchronization.

#### `semget`

```hopper
function semget(int key, int nsems, int semflg) int
```

Get or create a semaphore array. `key` is `0` (`IPC_PRIVATE`) for a private set. `nsems` is
the number of semaphores in the array. `semflg` is a bitmask: `512` (`IPC_CREAT`), `1024`
(`IPC_EXCL`), plus permission bits.

Returns a semaphore set identifier (non-negative) or a negative errno.

#### `semop`

```hopper
function semop(int semid, address sops, int nsops) int
```

Perform operations on one or more semaphores in the set. `sops` points to an array of
`nsops` `sembuf` structures:

| Offset | Type | Field | Meaning |
|--------|------|-------|---------|
| `0` | `u16` | `sem_num` | Semaphore index in the array |
| `2` | `i16` | `sem_op` | Operation: positive=V (increment), negative=P (decrement, blocking if result < 0), zero=wait for zero |
| `4` | `i16` | `sem_flg` | Flags: `IPC_NOWAIT=2048`, `SEM_UNDO=4096` |

Returns `0` on success or a negative errno.

#### `semctl`

```hopper
function semctl(int semid, int semnum, int cmd, int arg) int
```

Control or query a semaphore set. Common `cmd` values:

| Value | Constant | Meaning |
|-------|----------|---------|
| `0` | `IPC_RMID` | Remove the semaphore set |
| `2` | `IPC_STAT` | Get semaphore set metadata |
| `12` | `GETVAL` | Get value of semaphore `semnum` |
| `16` | `SETVAL` | Set value of semaphore `semnum` to `arg` |

### Process Execution

#### `execve`

```hopper
function execve(string filename, string[] argv, string[] envp) int
```

Replace the calling process image with the program at `filename`. `argv` is a null-terminated
array of argument strings. `envp` is a null-terminated array of environment strings in
`KEY=VALUE` form. On success, `execve` does not return. On error it returns a negative errno.

#### `execveat`

```hopper
function execveat(int dirfd, string path, string[] argv, string[] envp, int flags) int
```

Like `execve` but resolves `path` relative to `dirfd` (or the current directory if
`AtFlag.AT_FDCWD`).

#### `waitpid`

```hopper
function waitpid(int pid, address wstatus, int options) int
```

Wait for process `pid` to change state. `wstatus` receives the wait status (an opaque `int`
encoding exit code, signal, etc.). `options` is a bitmask of `WaitFlag` values:

| Constant | Value | Meaning |
|----------|-------|---------|
| `WaitFlag.WNOHANG` | `1` | Return immediately if no child has exited |
| `WaitFlag.WUNTRACED` | `2` | Also report stopped children |

Returns the PID of the changed child on success, `0` if `WNOHANG` and no child changed, or
a negative errno.

#### `waitid`

```hopper
function waitid(int idtype, int id, address infop, int options) int
```

Like `waitpid` but more expressive. `idtype` is an `IdType` value:

| Constant | Value | Meaning |
|----------|-------|---------|
| `IdType.P_ALL` | `0` | Wait for any child |
| `IdType.P_PID` | `1` | Wait for child with PID `id` |
| `IdType.P_PGID` | `2` | Wait for child in process group `id` |

`infop` receives a `siginfo_t` structure with full details.

### Signals

Signal delivery is the Linux mechanism for asynchronous inter-thread and inter-process
notification. The `Process` contract exposes the full signal API.

#### `kill`

```hopper
function kill(int pid, int sig) int
```

Send signal `sig` to process `pid`. Pass `-1` to send to all processes the caller has
permission to signal. `sig` is a `Signal` enum value.

#### `tkill`

```hopper
function tkill(int tid, int sig) int
```

Send signal `sig` to thread `tid` (thread-directed). Unlike `kill`, `tkill` targets a specific
thread within a thread group.

#### `tgkill`

```hopper
function tgkill(int tgid, int tid, int sig) int
```

Like `tkill` but requires the thread group ID for safety — avoids accidentally signaling a
recycled TID.

#### Signal Enum

```hopper
enum Signal {
    SIGHUP  =  1    // Hangup
    SIGINT  =  2    // Interrupt (Ctrl-C)
    SIGQUIT =  3    // Quit (Ctrl-\)
    SIGILL  =  4    // Illegal instruction
    SIGABRT =  6    // Abort
    SIGFPE  =  8    // Floating-point exception
    SIGKILL =  9    // Kill (cannot be caught or ignored)
    SIGSEGV = 11    // Segmentation fault
    SIGPIPE = 13    // Broken pipe
    SIGALRM = 14    // Alarm clock
    SIGTERM = 15    // Termination
    SIGUSR1 = 10    // User-defined signal 1
    SIGUSR2 = 12    // User-defined signal 2
    SIGCHLD = 17    // Child stopped or terminated
    SIGCONT = 18    // Continue if stopped
    SIGSTOP = 19    // Stop (cannot be caught or ignored)
    SIGTSTP = 20    // Terminal stop (Ctrl-Z)
}
```

#### `rt_sigaction`

```hopper
function rt_sigaction(int sig, address act, address oldact, int sigsetsize) int
```

Install a signal handler for `sig`. `act` and `oldact` point to `struct sigaction` structures.
`sigsetsize` MUST be `8` (the size of `sigset_t` on x86-64). Pass `0` for `oldact` to discard
the previous handler.

#### `rt_sigprocmask`

```hopper
function rt_sigprocmask(int how, address set, address oldset, int sigsetsize) int
```

Block, unblock, or set the signal mask for the calling thread. `how` is a `SigHow` value:

| Constant | Value | Meaning |
|----------|-------|---------|
| `SigHow.SIG_BLOCK` | `0` | Add `set` to the current mask |
| `SigHow.SIG_UNBLOCK` | `1` | Remove `set` from the current mask |
| `SigHow.SIG_SETMASK` | `2` | Replace the mask with `set` |

#### `rt_sigpending`

```hopper
function rt_sigpending(address set, int sigsetsize) int
```

Return the set of pending signals that are blocked. `set` receives the signal mask.

#### `rt_sigsuspend`

```hopper
function rt_sigsuspend(address mask, int sigsetsize) int
```

Atomically replace the signal mask with `mask` and suspend the calling thread until a signal
is delivered. Always returns `-EINTR`.

#### `pause`

```hopper
function pause() int
```

Suspend the calling thread until any signal is delivered. Always returns `-EINTR`.

#### `alarm`

```hopper
function alarm(int seconds) int
```

Schedule a `SIGALRM` to be delivered to the calling process after `seconds` seconds. Returns
the number of seconds remaining on any previously-set alarm.

#### `sigaltstack`

```hopper
function sigaltstack(address ss, address oss) int
```

Set or get the alternate signal stack. Useful for handling `SIGSEGV` caused by stack overflow
without itself crashing.

#### `ptrace`

```hopper
function ptrace(int request, int pid, address addr, address data) int
```

Control a traced process. Used to implement debuggers and system call tracers. `request`
selects the operation (e.g., `PTRACE_ATTACH=16`, `PTRACE_CONT=7`, `PTRACE_SYSCALL=24`,
`PTRACE_GETREGS=12`). Returns `0` on success or a negative errno.

### exit

```hopper
function exit(int code)
function exit_group(int code)
```

`exit` terminates the calling thread. `exit_group` terminates all threads in the thread
group (the entire process). Programs MUST call `exit_group` to terminate cleanly rather than
`exit`, unless they intend to terminate only the current thread within a multi-threaded process.
