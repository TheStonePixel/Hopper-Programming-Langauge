## Filesystem

The `FileSystem` interface declares the Linux filesystem system calls. It is defined in
`modules/linux/interfaces/FileSystem.hop` and satisfied on x86-64 Linux by
`modules/x86_64/src/LinuxSyscalls.hop`.

---

### Importing FileSystem

In `hopper.json`:

```json
{
  "dependencies": {
    "linux": "0.1.0",
    "x86_64": "0.1.0"
  },
  "targets": {
    "host": {
      "FileSystem": {
        "from":           "linux",
        "interface":      "modules/linux/interfaces/FileSystem.hop",
        "implementation": "modules/x86_64/src/LinuxSyscalls.hop"
      }
    }
  }
}
```

In source:

```hopper
import FileSystem from linux
```

### Enums

#### `OpenFlag`

Flags for `open`, `openat`, `creat`:

| Constant | Value | Meaning |
|----------|-------|---------|
| `OpenFlag.O_RDONLY` | `0` | Open for reading only |
| `OpenFlag.O_WRONLY` | `1` | Open for writing only |
| `OpenFlag.O_RDWR` | `2` | Open for reading and writing |
| `OpenFlag.O_CREAT` | `64` | Create file if it does not exist |
| `OpenFlag.O_EXCL` | `128` | Error if file already exists (used with `O_CREAT`) |
| `OpenFlag.O_TRUNC` | `512` | Truncate file to zero length on open |
| `OpenFlag.O_APPEND` | `1024` | Write always appends to end of file |
| `OpenFlag.O_NONBLOCK` | `2048` | Non-blocking I/O |
| `OpenFlag.O_CLOEXEC` | `524288` | Close file descriptor on `exec` |
| `OpenFlag.O_DIRECTORY` | `65536` | Fail if path is not a directory |

Flags are combined with the `|` operator:

```hopper
int fd = open("out.txt", OpenFlag.O_WRONLY | OpenFlag.O_CREAT | OpenFlag.O_TRUNC, 420)
```

#### `SeekWhence`

Seek anchor for `lseek`:

| Constant | Value | Meaning |
|----------|-------|---------|
| `SeekWhence.SEEK_SET` | `0` | Offset from beginning of file |
| `SeekWhence.SEEK_CUR` | `1` | Offset from current position |
| `SeekWhence.SEEK_END` | `2` | Offset from end of file |

#### `AtFlag`

Special file descriptor value for `*at` syscalls:

| Constant | Value | Meaning |
|----------|-------|---------|
| `AtFlag.AT_FDCWD` | `-100` | Use current working directory as base |

### Core File Operations

#### `open`

```hopper
function open(string path, int flags, int mode) int
```

Open the file at `path`. `flags` is a combination of `OpenFlag` values. `mode` specifies
permission bits (e.g., `0644` = `420` decimal) and is used only when `O_CREAT` is set.

Returns a non-negative file descriptor on success, or a negative errno on error.

```hopper
// Open a file for reading
int fd = open("data.bin", OpenFlag.O_RDONLY, 0)
if (fd < 0) {
    // error
}
```

#### `close`

```hopper
function close(int fd) int
```

Close file descriptor `fd`, releasing all associated resources. Returns `0` on success or a
negative errno.

Every file descriptor opened with `open`, `creat`, `dup`, `pipe`, or `socket` MUST eventually
be closed. Failing to close file descriptors is a resource leak.

```hopper
close(fd)
```

#### `creat`

```hopper
function creat(string path, int mode) int
```

Create or truncate a file. Equivalent to `open(path, O_WRONLY | O_CREAT | O_TRUNC, mode)`.
Returns a non-negative file descriptor or a negative errno.

#### `lseek`

```hopper
function lseek(int fd, int offset, int whence) int
```

Reposition the read/write offset of `fd`. `whence` is one of the `SeekWhence` constants.
Returns the new absolute offset from the beginning of the file, or a negative errno.

```hopper
int pos = lseek(fd, 0, SeekWhence.SEEK_END)   // seek to end; pos = file size
int _   = lseek(fd, 0, SeekWhence.SEEK_SET)   // seek back to beginning
```

### File Metadata

#### `stat`

```hopper
function stat(string path, address buf) int
```

Retrieve metadata for the file at `path` into the `stat` structure at `buf`. The structure
layout is the Linux `struct stat` (144 bytes on x86-64). Returns `0` on success or a negative
errno.

Key fields in the `stat` structure at the following byte offsets (x86-64 Linux ABI):

| Offset | Type | Field | Meaning |
|--------|------|-------|---------|
| `0` | `i64` | `st_dev` | Device ID |
| `8` | `i64` | `st_ino` | Inode number |
| `16` | `i32` | `st_mode` | File type and permissions |
| `20` | `i32` | `st_nlink` | Number of hard links |
| `24` | `i32` | `st_uid` | Owner user ID |
| `28` | `i32` | `st_gid` | Owner group ID |
| `40` | `i64` | `st_size` | File size in bytes |
| `88` | `i64` | `st_mtime` | Last modification time (seconds since epoch) |

Programs MUST read fields using `address` arithmetic:

```hopper
address statBuf = allocate 144
int rc = stat("/etc/hostname", statBuf)
// read st_size at offset 40
address sizePtr = statBuf + 40
// (read as 8-byte int via pointer cast — implementation-specific)
```

#### `fstat`

```hopper
function fstat(int fd, address buf) int
```

Like `stat` but operates on an open file descriptor `fd` rather than a path. Useful to stat
a file that has already been opened.

#### `lstat`

```hopper
function lstat(string path, address buf) int
```

Like `stat` but does not follow symbolic links — returns metadata for the symlink itself
rather than its target.

#### `statfs`

```hopper
function statfs(string path, address buf) int
```

Return filesystem statistics for the filesystem containing `path`. The `statfs` structure
describes total/free blocks and inodes.

#### `fstatfs`

```hopper
function fstatfs(int fd, address buf) int
```

Like `statfs` but takes an open file descriptor.

#### `access`

```hopper
function access(string path, int mode) int
```

Check whether the calling process can access `path`. `mode` is a bitmask: `0` (F_OK, existence),
`1` (X_OK, execute), `2` (W_OK, write), `4` (R_OK, read). Returns `0` if all requested
permissions are granted, or a negative errno.

### Permissions and Ownership

#### `chmod`

```hopper
function chmod(string path, int mode) int
```

Change the permission bits of `path` to `mode`. Returns `0` or a negative errno.

#### `fchmod`

```hopper
function fchmod(int fd, int mode) int
```

Change the permission bits of the open file `fd`.

#### `chown`

```hopper
function chown(string path, int uid, int gid) int
```

Change owner and group of `path`. Pass `-1` for `uid` or `gid` to leave it unchanged.

#### `fchown`

```hopper
function fchown(int fd, int uid, int gid) int
```

Change owner and group of the open file `fd`.

#### `lchown`

```hopper
function lchown(string path, int uid, int gid) int
```

Like `chown` but does not follow symbolic links.

#### `umask`

```hopper
function umask(int mask) int
```

Set the process file-creation mode mask to `mask`. Returns the previous umask. The umask is
subtracted from permission bits on every `open`, `mkdir`, and similar call.

#### `truncate`

```hopper
function truncate(string path, int length) int
```

Truncate or extend `path` to exactly `length` bytes. If the file is shorter, it is extended
with null bytes. Returns `0` or a negative errno.

#### `ftruncate`

```hopper
function ftruncate(int fd, int length) int
```

Like `truncate` but takes an open, writable file descriptor.

### Directories

#### `mkdir`

```hopper
function mkdir(string path, int mode) int
```

Create the directory `path` with permission bits `mode`. The `mode` is modified by the
process umask. Returns `0` or a negative errno. Does not create intermediate directories.

#### `rmdir`

```hopper
function rmdir(string path) int
```

Remove the empty directory `path`. Returns `0` or a negative errno. Fails with `-ENOTEMPTY`
if the directory contains any entries.

#### `getcwd`

```hopper
function getcwd(address buf, int bufsz) int
```

Copy the absolute pathname of the current working directory into `buf`. `bufsz` is the size
of `buf`. Returns `0` on success or a negative errno. `buf` must be large enough for the full
path including the null terminator.

```hopper
address cwdBuf = allocate 4096
int rc = getcwd(cwdBuf, 4096)
```

#### `chdir`

```hopper
function chdir(string path) int
```

Change the current working directory to `path`. Returns `0` or a negative errno.

#### `fchdir`

```hopper
function fchdir(int fd) int
```

Change the current working directory to the open directory `fd`.

#### `getdents64`

```hopper
function getdents64(int fd, address buf, int count) int
```

Read directory entries from the directory file descriptor `fd` into `buf`. Each entry is a
variable-length `linux_dirent64` structure. Returns the total bytes read, `0` at end of
directory, or a negative errno.

The `linux_dirent64` layout:

| Offset | Type | Field | Meaning |
|--------|------|-------|---------|
| `0` | `i64` | `d_ino` | Inode number |
| `8` | `i64` | `d_off` | Offset to next entry |
| `16` | `i16` | `d_reclen` | Total length of this entry (variable) |
| `18` | `i8` | `d_type` | File type (`4`=dir, `8`=regular, `10`=symlink) |
| `19` | `i8[]` | `d_name` | Null-terminated filename |

### File Operations

#### `rename`

```hopper
function rename(string oldpath, string newpath) int
```

Atomically rename `oldpath` to `newpath`. If `newpath` exists it is replaced. Both paths MUST
be on the same filesystem. Returns `0` or a negative errno.

#### `link`

```hopper
function link(string oldpath, string newpath) int
```

Create a hard link at `newpath` pointing to the same inode as `oldpath`. Returns `0` or a
negative errno.

#### `unlink`

```hopper
function unlink(string path) int
```

Remove the directory entry `path`. If this is the last hard link to the file and no open file
descriptors reference it, the file is deleted. Returns `0` or a negative errno.

#### `symlink`

```hopper
function symlink(string target, string linkpath) int
```

Create a symbolic link at `linkpath` pointing to `target`. Returns `0` or a negative errno.

#### `readlink`

```hopper
function readlink(string path, address buf, int bufsz) int
```

Read the target of the symbolic link `path` into `buf`. Does NOT null-terminate the result.
Returns the number of bytes placed in `buf`, or a negative errno.

### File Descriptor Duplication

#### `dup`

```hopper
function dup(int fd) int
```

Create a new file descriptor that refers to the same open file description as `fd`. The new
file descriptor is the lowest available number. Returns the new fd or a negative errno.

#### `dup2`

```hopper
function dup2(int oldfd, int newfd) int
```

Create a copy of `oldfd` as file descriptor number `newfd`, closing `newfd` first if it is
already open. Returns `newfd` or a negative errno.

#### `dup3`

```hopper
function dup3(int oldfd, int newfd, int flags) int
```

Like `dup2` but accepts `flags`. The only supported flag is `O_CLOEXEC` (`524288`).

---

### Pipes

#### `pipe`

```hopper
function pipe(address fds) int
```

Create a pipe and store the two file descriptors into the `int[2]` array at `fds`. `fds[0]`
is the read end; `fds[1]` is the write end. Returns `0` or a negative errno.

```hopper
address fds = allocate 16          // room for two i64 fds
int rc = pipe(fds)
int readEnd  = fds::value          // fds[0]
// fds[1] at offset 8 — read via address arithmetic
```

#### `pipe2`

```hopper
function pipe2(address fds, int flags) int
```

Like `pipe` but accepts `flags`. Supported flags: `O_CLOEXEC` and `O_NONBLOCK`.

---

### Synchronization

#### `fsync`

```hopper
function fsync(int fd) int
```

Flush all dirty buffers for `fd` to disk. Blocks until the device acknowledges the write.
Returns `0` or a negative errno.

#### `fdatasync`

```hopper
function fdatasync(int fd) int
```

Like `fsync` but does not flush metadata (timestamps, etc.) unless required for correctness.
Faster than `fsync` for data-only durability guarantees.

#### `sync`

```hopper
function sync() int
```

Flush all dirty buffers for all filesystems. Returns `0`.

---

### File Control

#### `fcntl`

```hopper
function fcntl(int fd, int cmd, int arg) int
```

Perform a control operation on `fd`. Common commands:

| `cmd` | Value | `arg` | Effect |
|-------|-------|-------|--------|
| `F_GETFL` | `3` | ignored | Get file status flags |
| `F_SETFL` | `4` | flags | Set file status flags |
| `F_GETFD` | `1` | ignored | Get file descriptor flags |
| `F_SETFD` | `2` | flags | Set file descriptor flags |
| `F_DUPFD_CLOEXEC` | `1030` | min fd | Duplicate with O_CLOEXEC |

Returns value on success or a negative errno.

#### `flock`

```hopper
function flock(int fd, int operation) int
```

Apply or remove an advisory lock on `fd`. `operation` values: `1` (LOCK_SH, shared),
`2` (LOCK_EX, exclusive), `4` (LOCK_NB, non-blocking), `8` (LOCK_UN, unlock).
Returns `0` or a negative errno.

### `*at` Variants

The `*at` family of syscalls takes a directory file descriptor as a base for resolving relative
paths, which allows race-condition-free filesystem operations (no TOCTOU).

| Function | Base variant |
|----------|-------------|
| `openat(int dirfd, string path, int flags, int mode)` | `open` |
| `mkdirat(int dirfd, string path, int mode)` | `mkdir` |
| `unlinkat(int dirfd, string path, int flags)` | `unlink` or `rmdir` |
| `renameat(int old_dirfd, string oldpath, int new_dirfd, string newpath)` | `rename` |
| `linkat(int old_dirfd, string oldpath, int new_dirfd, string newpath, int flags)` | `link` |
| `symlinkat(string target, int new_dirfd, string linkpath)` | `symlink` |
| `readlinkat(int dirfd, string path, address buf, int bufsz)` | `readlink` |
| `fchmodat(int dirfd, string path, int mode, int flags)` | `chmod` |
| `faccessat(int dirfd, string path, int mode, int flags)` | `access` |
| `fchownat(int dirfd, string path, int uid, int gid, int flags)` | `chown` |

Pass `AtFlag.AT_FDCWD` (`-100`) as `dirfd` to resolve relative to the current working
directory, giving the same behavior as the non-`at` variant.

### Filesystem Watches (inotify)

#### `inotify_init1`

```hopper
function inotify_init1(int flags) int
```

Create a new inotify instance. `flags`: `0` or `O_CLOEXEC` (`524288`) or `O_NONBLOCK`
(`2048`). Returns an inotify file descriptor or a negative errno.

#### `inotify_add_watch`

```hopper
function inotify_add_watch(int fd, string path, int mask) int
```

Add a watch for `path` to the inotify instance `fd`. `mask` is a bitmask of events
(e.g., `IN_CREATE=256`, `IN_DELETE=512`, `IN_MODIFY=2`, `IN_CLOSE_WRITE=8`). Returns
a watch descriptor (non-negative) or a negative errno.

#### `inotify_rm_watch`

```hopper
function inotify_rm_watch(int fd, int wd) int
```

Remove the watch `wd` from inotify instance `fd`. Returns `0` or a negative errno.

### Typical File Read Pattern

```hopper
import IO from linux
import FileSystem from linux

entry main {
    int fd = open("input.txt", OpenFlag.O_RDONLY, 0)
    if (fd < 0) {
        write(StdFd.STDERR, cast<address>("open failed\n"), 12)
        exit(1)
    }

    address buf = allocate 4096
    int n = read(fd, buf, 4096)
    while (n > 0) {
        write(StdFd.STDOUT, buf, n)
        n = read(fd, buf, 4096)
    }

    close(fd)
    deallocate buf
}
```
