## Math

**No conforming math module exists.** A legacy `math` module is present in the repository
but has not been migrated to the current contract standard and is not part of the Hopper
standard library. Programs MUST NOT import from the nonconforming `math` module.

---

### What IS Available Without a Module

The following math operations are built into the compiler and require no import:

#### Integer Arithmetic

All standard arithmetic operators work on `int`, `byte`, `unsigned int`, and `unsigned byte`:

| Operator | Operation | Notes |
|----------|-----------|-------|
| `a + b` | Addition | Wraps on overflow (two's complement) |
| `a - b` | Subtraction | Wraps on underflow |
| `a * b` | Multiplication | Wraps on overflow |
| `a / b` | Division | Truncates toward zero; undefined behavior if `b == 0` |
| `a % b` | Remainder | Sign follows the dividend; undefined if `b == 0` |
| `-a` | Negation | |
| `a << b` | Left shift | Undefined behavior if `b >= 64` |
| `a >> b` | Right shift | Arithmetic (sign-extending) for `int`; logical for `unsigned int` |
| `a & b` | Bitwise AND | |
| `a \| b` | Bitwise OR | |
| `a ^ b` | Bitwise XOR | |
| `~a` | Bitwise NOT | |

Division by zero is undefined behavior. The compiler does not insert a runtime check.

#### Float Arithmetic

All standard arithmetic operators work on `float` (64-bit IEEE 754):

| Operator | Operation |
|----------|-----------|
| `a + b` | Addition |
| `a - b` | Subtraction |
| `a * b` | Multiplication |
| `a / b` | Division (division by zero produces infinity per IEEE 754) |
| `-a` | Negation |

Comparison operators (`==`, `!=`, `<`, `<=`, `>`, `>=`) follow IEEE 754 ordered comparison
semantics. Comparing with NaN always produces `false` for ordered comparisons.

#### Mixed-Type Arithmetic

Hopper does not perform implicit type coercion. To combine `int` and `float`:

```hopper
import cast from cast

float x = 3.14
int   n = 2
float result = x * cast<float>(n)
```

### Transcendental Functions

**Not yet implemented.**

There are no `sin`, `cos`, `tan`, `sqrt`, `log`, `exp`, `pow`, or similar transcendental
functions in the standard library.

Programs that require these functions MUST use `extern` declarations to libc:

```hopper
extern function sqrt(float x) float
extern function sin(float x) float
extern function cos(float x) float
extern function log(float x) float
extern function exp(float x) float
extern function pow(float base, float exp) float
extern function fabs(float x) float
extern function floor(float x) float
extern function ceil(float x) float
```

Linking with libc (`-lm`) is required when using these extern declarations. This is a
temporary workaround â€” extern declarations to libc MUST NOT appear in conforming module code.
Once a Hopper math module is standardized, programs SHOULD migrate to it.

### Integer Math Utilities

The following patterns implement common integer math operations without any import:

#### Absolute value (integer)

```hopper
function abs(int x) int {
    if (x < 0) {
        return -x
    }
    return x
}
```

#### Minimum / Maximum

```hopper
function min(int a, int b) int {
    if (a < b) {
        return a
    }
    return b
}

function max(int a, int b) int {
    if (a > b) {
        return a
    }
    return b
}
```

#### Integer power

```hopper
function ipow(int base, int exp) int {
    int result = 1
    while (exp > 0) {
        result = result * base
        exp = exp - 1
    }
    return result
}
```

#### Clamp

```hopper
function clamp(int x, int lo, int hi) int {
    if (x < lo) { return lo }
    if (x > hi) { return hi }
    return x
}
```

### Bit Manipulation

SIMD-based bit operations are available through the `x86_64` SIMD module (Â§13.9):

- `popcnt16(int mask) int` â€” count set bits in a 16-bit mask using the `popcnt` instruction
- `bsf16(int mask) int` â€” index of lowest set bit using the `bsf` instruction

For scalar bit manipulation on `int`:

```hopper
// Count trailing zeros (CTZ) â€” using inline asm in a module file
function ctz(int x) int {
    int n = 0
    asm {
        rdi = x
        bsf rax, rdi
        n = rax
    }
    return n
}
```

Inline assembly for bit manipulation MUST be placed in module files within the `x86_64/`
module â€” not in program source files.

### Math Module Roadmap

A conforming `math/` module is planned. When implemented it will provide at minimum:

- Integer: `abs`, `min`, `max`, `clamp`, `gcd`, `lcm`, integer square root
- Float: `abs`, `floor`, `ceil`, `round`, `sqrt`, `min`, `max`, `clamp`
- Transcendentals: `sin`, `cos`, `tan`, `asin`, `acos`, `atan`, `atan2`, `log`, `log2`,
  `log10`, `exp`, `exp2`, `pow`
- Constants: `PI`, `E`, `TAU`, `INF`, `NAN`

The implementation MUST NOT delegate to libc â€” it will use Hopper source with SSE2/AVX
intrinsics via the `x86_64/` SIMD module for performance-critical paths.
