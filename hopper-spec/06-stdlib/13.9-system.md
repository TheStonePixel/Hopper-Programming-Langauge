## System and Memory

This section covers two interfaces:

- **`System`** (`modules/linux/contracts/System.hop`) — system information, clocks,
  scheduling, resource limits, and miscellaneous kernel interfaces.
- **`Memory`** (`modules/linux/contracts/Memory.hop`) — virtual memory mapping,
  protection, and locking.

Additionally, the `x86_64` SIMD capability module is documented here, as it is an ISA-level
hardware capability rather than an OS contract.

---

### Importing System and Memory

In `hopper.json`:

```json
{
  "dependencies": {
    "linux": "0.1.0",
    "x86_64": "0.1.0"

  "targets": {
    "host": {
      "System": {
        "from":           "linux",

```

### System Interface

#### System Information

##### `uname`

```hopper
function uname(address buf) int
```

Fill the `utsname` structure at `buf` with kernel and machine information. On x86-64 Linux
the structure is six null-padded 65-byte fields:

| Offset | Field | Meaning |
|--------|-------|---------|
| `0` | `sysname` | OS name (e.g., `"Linux"`) |
| `65` | `nodename` | Hostname |
| `130` | `release` | Kernel release (e.g., `"6.1.0"`) |
| `195` | `version` | Kernel version string |
| `260` | `machine` | Hardware type (e.g., `"x86_64"`) |
| `325` | `domainname` | NIS domain name |

Total size: 390 bytes. Returns `0` or a negative errno.

```hopper
address utsBuf = allocate 390
int rc = uname(utsBuf)
// utsBuf now contains null-terminated strings at the offsets above
```

##### `sysinfo`

```hopper
function sysinfo(address buf) int
```

Fill the `sysinfo` structure at `buf` with system-wide statistics. Key fields on x86-64:

| Offset | Type | Field | Meaning |
|--------|------|-------|---------|
| `0` | `i64` | `uptime` | Seconds since boot |
| `8` | `i64[3]` | `loads` | 1-, 5-, 15-minute load averages (scaled by 65536) |
| `32` | `i64` | `totalram` | Total physical memory (in `mem_unit` units) |
| `40` | `i64` | `freeram` | Available physical memory |
| `48` | `i64` | `sharedram` | Amount of shared memory |
| `56` | `i64` | `bufferram` | Buffer cache memory |
| `64` | `i64` | `totalswap` | Total swap space |
| `72` | `i64` | `freeswap` | Available swap |
| `80` | `i16` | `procs` | Number of current processes |
| `88` | `i64` | `totalhigh` | Total high memory |
| `96` | `i64` | `freehigh` | Available high memory |
| `104` | `i32` | `mem_unit` | Memory unit size in bytes (usually 1) |

Returns `0` or a negative errno.

#### Clocks and Time

##### `ClockId` Enum

| Constant | Value | Meaning |
|----------|-------|---------|
| `ClockId.CLOCK_REALTIME` | `0` | Wall clock time (UTC). Affected by `settimeofday`. |
| `ClockId.CLOCK_MONOTONIC` | `1` | Monotonically increasing since boot. Not affected by time adjustments. |
| `ClockId.CLOCK_PROCESS_CPUTIME` | `2` | CPU time consumed by the calling process. |
| `ClockId.CLOCK_BOOTTIME` | `7` | Like `CLOCK_MONOTONIC` but includes time the system was suspended. |

##### `clock_gettime`

```hopper
function clock_gettime(int clockid, address ts) int
```

Read the clock identified by `clockid` into a `timespec` structure at `ts`:

| Offset | Type | Field | Meaning |
|--------|------|-------|---------|
| `0` | `i64` | `tv_sec` | Seconds |
| `8` | `i64` | `tv_nsec` | Nanoseconds (0–999999999) |

Returns `0` or a negative errno.

```hopper
address ts = allocate 16
int rc = clock_gettime(ClockId.CLOCK_MONOTONIC, ts)
// Read seconds: ts at offset 0 (via address arithmetic)
```

##### `clock_settime`

```hopper
function clock_settime(int clockid, address ts) int
```

Set the clock identified by `clockid` to the time in `ts`. Requires `CAP_SYS_TIME`.

##### `clock_getres`

```hopper
function clock_getres(int clockid, address res) int
```

Query the resolution (precision) of `clockid`. Writes a `timespec` with the minimum
measurable time increment.

##### `clock_nanosleep`

```hopper
function clock_nanosleep(int clockid, int flags, address req, address rem) int
```

Sleep for the duration (or until the time) specified in `req` relative to `clockid`.
`flags`: `0` for relative sleep, `1` (`TIMER_ABSTIME`) for absolute time sleep.
`rem` receives the remaining time if interrupted; may be `0`.

Returns `0`, `-EINTR` (interrupted by signal), or another negative errno.

##### `nanosleep`

```hopper
function nanosleep(address req, address rem) int
```

Sleep for the duration in `req` (a `timespec`). Equivalent to
`clock_nanosleep(CLOCK_REALTIME, 0, req, rem)`. Returns `0` or `-EINTR`.

##### `gettimeofday`

```hopper
function gettimeofday(address tv, address tz) int
```

Read the current time into a `timeval` structure at `tv` (microsecond resolution):

| Offset | Type | Field | Meaning |
|--------|------|-------|---------|
| `0` | `i64` | `tv_sec` | Seconds since epoch |
| `8` | `i64` | `tv_usec` | Microseconds |

Pass `0` for `tz` (deprecated). Returns `0` or a negative errno.

Prefer `clock_gettime(CLOCK_REALTIME, ...)` for new code — it provides nanosecond resolution.

##### `settimeofday`

```hopper
function settimeofday(address tv, address tz) int
```

Set the system clock. Requires `CAP_SYS_TIME`. Pass `0` for `tz`.

##### `times`

```hopper
function times(address buf) int
```

Fill a `tms` structure with process and child CPU time accounting:

| Offset | Type | Field | Meaning |
|--------|------|-------|---------|
| `0` | `i64` | `tms_utime` | User CPU time of calling process (in clock ticks) |
| `8` | `i64` | `tms_stime` | System CPU time of calling process |
| `16` | `i64` | `tms_cutime` | User CPU time of waited-for children |
| `24` | `i64` | `tms_cstime` | System CPU time of waited-for children |

Returns elapsed real time in clock ticks or a negative errno.

#### Interval Timers

##### `ItimerWhich` Enum

| Constant | Value | Meaning |
|----------|-------|---------|
| `ItimerWhich.ITIMER_REAL` | `0` | Real time; delivers `SIGALRM` |
| `ItimerWhich.ITIMER_VIRTUAL` | `1` | Process virtual time; delivers `SIGVTALRM` |
| `ItimerWhich.ITIMER_PROF` | `2` | Process + system time; delivers `SIGPROF` |

##### `getitimer`

```hopper
function getitimer(int which, address val) int
```

Query an interval timer. `val` receives a `struct itimerval` with two `timeval` fields:
`it_interval` (repeat period) and `it_value` (time until next delivery).

##### `setitimer`

```hopper
function setitimer(int which, address val, address oval) int
```

Set an interval timer. `val` specifies the new timer. `oval` (may be `0`) receives the
previous timer settings. Setting `it_value` to zero disarms the timer.

#### Resource Limits

##### `Rlimit` Enum

| Constant | Value | Resource |
|----------|-------|---------|
| `Rlimit.RLIMIT_CORE` | `4` | Maximum size of core dump file |
| `Rlimit.RLIMIT_STACK` | `3` | Maximum stack size |
| `Rlimit.RLIMIT_NOFILE` | `7` | Maximum number of open file descriptors |
| `Rlimit.RLIMIT_AS` | `9` | Maximum virtual memory (address space) size |

##### `getrlimit`

```hopper
function getrlimit(int resource, address rlim) int
```

Query the soft and hard limits for `resource`. `rlim` receives a `struct rlimit` (two `i64`
values): `rlim_cur` (soft limit) at offset `0` and `rlim_max` (hard limit) at offset `8`.

##### `setrlimit`

```hopper
function setrlimit(int resource, address rlim) int
```

Set the soft and hard limits for `resource`. Processes may lower limits freely; raising the
hard limit requires `CAP_SYS_RESOURCE`.

#### `getrusage`

```hopper
function getrusage(int who, address rusage) int
```

Get resource usage statistics. `who` is a `RusageWho` value:

| Constant | Value | Meaning |
|----------|-------|---------|
| `RusageWho.RUSAGE_SELF` | `0` | Calling process |
| `RusageWho.RUSAGE_CHILDREN` | `-1` | All terminated children |

`rusage` receives a `struct rusage` (144 bytes). Key fields:

| Offset | Type | Field | Meaning |
|--------|------|-------|---------|
| `0` | `timeval` (16 bytes) | `ru_utime` | User CPU time used |
| `16` | `timeval` (16 bytes) | `ru_stime` | System CPU time used |
| `32` | `i64` | `ru_maxrss` | Maximum resident set size (KB) |
| `48` | `i64` | `ru_ixrss` | Integral shared memory size |
| `72` | `i64` | `ru_minflt` | Minor page faults (no I/O required) |
| `80` | `i64` | `ru_majflt` | Major page faults (I/O required) |
| `104` | `i64` | `ru_nvcsw` | Voluntary context switches |
| `112` | `i64` | `ru_nivcsw` | Involuntary context switches |

#### Process Control

##### `prctl`

```hopper
function prctl(int option, int arg2, int arg3, int arg4, int arg5) int
```

Process control. `option` selects the operation. Common uses:

| Option | Value | Meaning |
|--------|-------|---------|
| `PR_SET_NAME` | `15` | Set thread name (arg2 is a pointer to a 16-byte string) |
| `PR_GET_NAME` | `16` | Get thread name |
| `PR_SET_DUMPABLE` | `4` | Control core dump generation |
| `PR_SET_NO_NEW_PRIVS` | `38` | Prevent gaining new privileges via execve |

Returns `0` or the queried value on success, or a negative errno.

##### `arch_prctl`

```hopper
function arch_prctl(int code, address addr) int
```

Architecture-specific control. On x86-64, used to set the `FS` and `GS` base registers for
thread-local storage:

| Code | Value | Meaning |
|------|-------|---------|
| `ARCH_SET_GS` | `0x1001` | Set GS base register |
| `ARCH_SET_FS` | `0x1002` | Set FS base register (thread-local storage pointer) |
| `ARCH_GET_FS` | `0x1003` | Get FS base register |
| `ARCH_GET_GS` | `0x1004` | Get GS base register |

##### `brk`

```hopper
function brk(address addr) int
```

Set the program break (top of the heap data segment). Passing `0` returns the current break
address. Low-level heap expansion without using `mmap`. Prefer `allocate` / `deallocate` or
`mmap` for new code.

##### `chroot`

```hopper
function chroot(string path) int
```

Change the root directory of the calling process to `path`. Requires `CAP_SYS_CHROOT`.
Returns `0` or a negative errno.

### Memory Interface

#### Enums

##### `Prot`

Memory protection flags for `mmap` and `mprotect`:

| Constant | Value | Meaning |
|----------|-------|---------|
| `Prot.PROT_NONE` | `0` | No access (guard page) |
| `Prot.PROT_READ` | `1` | Pages may be read |
| `Prot.PROT_WRITE` | `2` | Pages may be written |
| `Prot.PROT_EXEC` | `4` | Pages may be executed |

Combine with `|`: `Prot.PROT_READ | Prot.PROT_WRITE`.

##### `MapFlag`

Flags for `mmap`:

| Constant | Value | Meaning |
|----------|-------|---------|
| `MapFlag.MAP_SHARED` | `1` | Share changes with other processes mapping the same file |
| `MapFlag.MAP_PRIVATE` | `2` | Copy-on-write mapping; changes are process-private |
| `MapFlag.MAP_FIXED` | `16` | Place the mapping at exactly `addr`; replaces any existing mapping |
| `MapFlag.MAP_ANON` | `32` | Anonymous mapping (not backed by any file) |

`MAP_PRIVATE | MAP_ANON` is the standard way to allocate large memory regions without a file.

##### `MsyncFlag`

Flags for `msync`:

| Constant | Value | Meaning |
|----------|-------|---------|
| `MsyncFlag.MS_ASYNC` | `1` | Initiate write-back but do not wait |
| `MsyncFlag.MS_INVALIDATE` | `2` | Invalidate cached copies in other mappings |
| `MsyncFlag.MS_SYNC` | `4` | Wait for write-back to complete |

##### `MadviseAdvice`

Hints for `madvise`:

| Constant | Value | Meaning |
|----------|-------|---------|
| `MadviseAdvice.MADV_NORMAL` | `0` | Default behavior |
| `MadviseAdvice.MADV_RANDOM` | `1` | Expect random access (disable read-ahead) |
| `MadviseAdvice.MADV_SEQUENTIAL` | `2` | Expect sequential access (aggressive read-ahead) |
| `MadviseAdvice.MADV_DONTNEED` | `4` | Do not need these pages (kernel may free them) |
| `MadviseAdvice.MADV_FREE` | `8` | Pages are free to reuse; contents may be discarded |

##### `MlockFlag`

Flags for `mlockall`:

| Constant | Value | Meaning |
|----------|-------|---------|
| `MlockFlag.MCL_CURRENT` | `1` | Lock all currently mapped pages |
| `MlockFlag.MCL_FUTURE` | `2` | Lock all pages mapped in the future |

##### `MemfdFlag`

Flags for `memfd_create`:

| Constant | Value | Meaning |
|----------|-------|---------|
| `MemfdFlag.MFD_CLOEXEC` | `1` | Set close-on-exec on the returned fd |
| `MemfdFlag.MFD_ALLOW_SEALING` | `2` | Allow sealing operations via `fcntl` |

#### Memory Function Reference

##### `mmap`

```hopper
function mmap(address addr, int len, int prot, int flags, int fd, int offset) address
```

Map `len` bytes into the virtual address space. `addr` is a hint for placement (pass `0` to
let the kernel choose). `prot` is a `Prot` bitmask. `flags` is a `MapFlag` bitmask. For
anonymous mappings, `fd` MUST be `-1` and `offset` MUST be `0`.

Returns the mapped address on success, or `-1` cast to `address` (i.e., `0xFFFFFFFFFFFFFFFF`)
on error. Programs MUST check for the error return value.

```hopper
// Allocate 1 MB of anonymous memory
address mem = mmap(0, 1048576,
    Prot.PROT_READ | Prot.PROT_WRITE,
    MapFlag.MAP_PRIVATE | MapFlag.MAP_ANON,
    -1, 0)
```

##### `munmap`

```hopper
function munmap(address addr, int len) int
```

Unmap `len` bytes starting at `addr`. `addr` MUST be page-aligned. `len` is rounded up to
the next page boundary by the kernel. Returns `0` or a negative errno.

```hopper
munmap(mem, 1048576)
```

##### `mprotect`

```hopper
function mprotect(address addr, int len, int prot) int
```

Change the protection of `len` bytes at `addr`. `addr` MUST be page-aligned. `prot` is a
`Prot` bitmask. Returns `0` or a negative errno.

```hopper
// Make a region read-only after writing
mprotect(mem, pageSize, Prot.PROT_READ)
```

##### `mremap`

```hopper
function mremap(address old_addr, int old_size, int new_size, int flags, address new_addr) address
```

Resize an existing mapping from `old_size` to `new_size`. `flags`: `1` (`MREMAP_MAYMOVE`)
allows the kernel to move the mapping; `3` (`MREMAP_MAYMOVE | MREMAP_FIXED`) moves to
`new_addr`. Returns the new address or the error sentinel.

##### `msync`

```hopper
function msync(address addr, int len, int flags) int
```

Flush changes in a file-backed mapping to the underlying file. `flags` is a `MsyncFlag`
bitmask. Returns `0` or a negative errno.

##### `madvise`

```hopper
function madvise(address addr, int len, int advice) int
```

Give the kernel a hint about future access patterns for the region. `advice` is a
`MadviseAdvice` value. Returns `0` or a negative errno.

`MADV_DONTNEED` is commonly used to return pages to the OS after a bulk operation without
unmapping the virtual address range:

```hopper
madvise(buf, bufSize, MadviseAdvice.MADV_DONTNEED)
```

##### `mincore`

```hopper
function mincore(address addr, int len, address vec) int
```

Query which pages of a mapping are resident in RAM. `vec` is a byte array; each byte
corresponds to one page; bit `0` of each byte is set if the page is resident.
Returns `0` or a negative errno.

##### `mlock`

```hopper
function mlock(address addr, int len) int
```

Lock `len` bytes at `addr` into RAM, preventing them from being paged out. Requires
`CAP_IPC_LOCK` or a non-zero `RLIMIT_MEMLOCK`. Returns `0` or a negative errno.

##### `munlock`

```hopper
function munlock(address addr, int len) int
```

Unlock previously locked pages. Returns `0` or a negative errno.

##### `mlockall`

```hopper
function mlockall(int flags) int
```

Lock all mapped pages of the calling process into RAM. `flags` is an `MlockFlag` bitmask.
Used in real-time programs to eliminate page-fault latency. Returns `0` or a negative errno.

##### `munlockall`

```hopper
function munlockall() int
```

Unlock all locked pages. Returns `0`.

##### `memfd_create`

```hopper
function memfd_create(string name, int flags) int
```

Create an anonymous file descriptor backed entirely by memory. `name` is a label for
debugging (visible in `/proc/<pid>/fd/`). `flags` is a `MemfdFlag` bitmask.

Returns a file descriptor or a negative errno. The fd may be used with `mmap`, `ftruncate`,
and `sendfile`. It has no associated path and persists as long as the fd is open or a mapping
exists.

```hopper
int memfd = memfd_create("shared_buf", MemfdFlag.MFD_CLOEXEC)
ftruncate(memfd, 65536)                                    // set size
address mem = mmap(0, 65536,
    Prot.PROT_READ | Prot.PROT_WRITE,
    MapFlag.MAP_SHARED,
    memfd, 0)
```

### 13.9.3 SIMD Capability Module (`x86_64`)

The SIMD module provides SSE2 vector instructions for byte scanning and transformation.
It is an ISA capability — not an OS contract — so it is exposed as a class rather than free
functions.

**Importing:**

```json
{
  "targets": {
    "host": {
      "SIMD": {
        "from":           "x86_64",
        "contract":      "modules/x86_64/contracts/SIMD.hop",
        "implementation": "modules/x86_64/src/SIMD.hop"
      }
    }
  }
}
```

```hopper
import SIMD from x86_64
```

Programs instantiate `X86SIMD` directly:

```hopper
X86SIMD simd = X86SIMD()
```

#### SIMD Interface

```hopper
contract SIMD {
    function scanByte16(address ptr, int b) int
    function scanByte2x16(address ptr, int b1, int b2) int
    function scanByteOR4x16(address ptr, int b1, int b2, int b3, int b4) int
    function popcnt16(int mask) int
    function bsf16(int mask) int
    function copy16(address src, address dst)
    function rangeXor16(address src, address dst, int lo, int hi, int xorVal)
    function translateRange(address src, address dst, int count, int lo, int hi, int xorVal) int
}
```

#### SIMD Function Reference

##### `scanByte16`

```hopper
function scanByte16(address ptr, int b) int
```

Load 16 bytes from `ptr` and compare each byte to `b`. Returns a 16-bit bitmask where bit `i`
is set if `ptr[i] == b`. Uses SSE2 `pcmpeqb` + `pmovmskb`.

`ptr` does not need to be aligned.

```hopper
// Find newline in a 16-byte window
int mask = simd.scanByte16(buf + i, 10)   // 10 = '\n'
if (mask != 0) {
    int offset = simd.bsf16(mask)         // position of first newline
}
```

##### `scanByte2x16`

```hopper
function scanByte2x16(address ptr, int b1, int b2) int
```

Single 16-byte load; compare against two different byte values simultaneously.
Returns a 32-bit integer: bits 0–15 are the mask for `b1`; bits 16–31 are the mask for `b2`.

```hopper
// Scan for both '\r' (13) and '\n' (10) in one load
int masks = simd.scanByte2x16(buf + i, 13, 10)
int crMask = masks & 0xFFFF
int nlMask = (masks >> 16) & 0xFFFF
```

##### `scanByteOR4x16`

```hopper
function scanByteOR4x16(address ptr, int b1, int b2, int b3, int b4) int
```

Single 16-byte load; OR together four byte comparisons. Returns a 16-bit mask where bit `i`
is set if `ptr[i]` equals any of `b1`, `b2`, `b3`, or `b4`.

```hopper
// Scan for any whitespace character: space, tab, \r, \n
int mask = simd.scanByteOR4x16(buf + i, 32, 9, 13, 10)
```

##### `popcnt16`

```hopper
function popcnt16(int mask) int
```

Count the number of set bits in the lower 16 bits of `mask`. Uses the `popcnt` instruction.

```hopper
// Count newlines in a 16-byte window
int nlMask = simd.scanByte16(buf + i, 10)
int count  = simd.popcnt16(nlMask)
```

##### `bsf16`

```hopper
function bsf16(int mask) int
```

Return the index (0–15) of the lowest set bit in `mask` using the `bsf` (Bit Scan Forward)
instruction. `mask` MUST be non-zero; calling with `mask == 0` is undefined behavior.

```hopper
if (mask != 0) {
    int pos = simd.bsf16(mask)
}
```

##### `copy16`

```hopper
function copy16(address src, address dst)
```

Copy exactly 16 bytes from `src` to `dst` using a single SIMD load/store. No alignment
requirement. Faster than a scalar byte loop for 16-byte copies.

##### `rangeXor16`

```hopper
function rangeXor16(address src, address dst, int lo, int hi, int xorVal)
```

Load 16 bytes from `src`. For each byte in the range `[lo, hi]` (unsigned comparison),
XOR it with `xorVal`. Store the result to `dst`. Useful for case conversion (e.g., ASCII
uppercase→lowercase: `lo=65, hi=90, xorVal=32`).

```hopper
// Convert 16 bytes of uppercase ASCII to lowercase in one SIMD operation
simd.rangeXor16(srcBuf + i, dstBuf + i, 65, 90, 32)
```

##### `translateRange`

```hopper
function translateRange(address src, address dst, int count, int lo, int hi, int xorVal) int
```

Apply the `rangeXor16` transformation across an entire buffer of `count` bytes. Processes
bytes in 16-byte SIMD chunks with a scalar tail for remaining bytes. Returns `count`.

```hopper
// Convert an entire buffer to lowercase
simd.translateRange(input, output, length, 65, 90, 32)
```

#### SIMD Usage Constraints

- SIMD operations MUST NOT be used in program source files directly. All `asm {}` blocks that
  use SSE2 instructions live inside `x86_64/src/SIMD.hop`. Programs call the `X86SIMD` class
  methods.
- `ptr` passed to `scanByte16` and `copy16` MUST point to readable memory of at least 16
  bytes, OR the caller MUST ensure the 16-byte window does not cross a page boundary into
  unmapped memory. A common pattern is to pad input buffers to the next 16-byte boundary.
- `bsf16` with a zero argument is undefined behavior. Always check `mask != 0` before
  calling.
