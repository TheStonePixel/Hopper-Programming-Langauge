## I/O

The `IO` contract declares the Linux I/O system calls. It is defined in
`modules/linux/contracts/IO.hop` and satisfied on x86-64 Linux by
`modules/x86_64/src/LinuxSyscalls.hop`.

---

### Importing IO

In `hopper.json`:

```json
{
  "dependencies": {
    "linux": "0.1.0",
    "x86_64": "0.1.0"
  },
  "targets": {
    "host": {
      "IO": {
        "from":           "linux",
        "contract":      "modules/linux/contracts/IO.hop",
        "implementation": "modules/x86_64/src/LinuxSyscalls.hop"
      }
    }
  }
}
```

In source:

```hopper
import IO from linux
```

After this import, all free functions declared in `LinuxSyscalls.hop` that satisfy the `IO`
contract are in scope — `read`, `write`, `pread64`, `pwrite64`, `readv`, `writev`,
`sendfile`, `splice`, `tee`, `ioctl`.

---

### Standard File Descriptor Constants

`IO.hop` defines the `StdFd` enum:

```hopper
enum StdFd {
    STDIN  = 0
    STDOUT = 1
    STDERR = 2
}
```

These constants MUST be used in preference to bare integer literals for standard I/O file
descriptors.

---

### Function Reference

#### `read`

```hopper
function read(int fd, address buf, int count) int
```

Read up to `count` bytes from file descriptor `fd` into the buffer at `buf`. Returns the
number of bytes actually read, `0` on end-of-file, or a negative errno value on error.

`buf` MUST point to a writable memory region of at least `count` bytes. The call may return
fewer bytes than requested (short read) even when data is available — the caller MUST loop if
it requires exactly `count` bytes.

```hopper
address buf = allocate 4096
int n = read(StdFd.STDIN, buf, 4096)
```

#### `write`

```hopper
function write(int fd, address buf, int count) int
```

Write up to `count` bytes from `buf` to file descriptor `fd`. Returns the number of bytes
written, or a negative errno value on error.

A short write (return value less than `count`) is possible for regular files when interrupted
by a signal, and is normal for non-blocking sockets. Callers MUST loop if they require all
bytes to be written.

**Hello, world — minimal pattern:**

```hopper
import IO from linux

entry main {
    string msg = "Hello, world!\n"
    write(StdFd.STDOUT, cast<address>(msg), 14)
}
```

`write` returns the number of bytes written. Programs MUST check the return value; ignoring it
is a potential data-loss bug.

#### `pread64`

```hopper
function pread64(int fd, address buf, int count, int offset) int
```

Read `count` bytes from `fd` beginning at file position `offset`, without changing the file's
seek position. Equivalent to `lseek` + `read` but atomic with respect to the file offset.
Useful for concurrent reads from multiple threads on the same file descriptor.

Returns bytes read, `0` on EOF, or a negative errno on error.

#### `pwrite64`

```hopper
function pwrite64(int fd, address buf, int count, int offset) int
```

Write `count` bytes from `buf` to `fd` at file position `offset`, without changing the file's
seek position.

Returns bytes written or a negative errno.

#### `readv`

```hopper
function readv(int fd, address iov, int iovcnt) int
```

Scatter read: read from `fd` into `iovcnt` buffers described by the `iovec` array at `iov`.
Each `iovec` is a struct `{ address iov_base; int iov_len }` (laid out according to the
Linux ABI). The kernel fills the buffers in order.

Returns total bytes read across all vectors, or a negative errno.

#### `writev`

```hopper
function writev(int fd, address iov, int iovcnt) int
```

Gather write: write `iovcnt` buffers described by the `iovec` array at `iov` to `fd` as a
single operation. Useful for writing protocol headers and payloads without copying them into a
single buffer first.

Returns total bytes written, or a negative errno.

#### `sendfile`

```hopper
function sendfile(int out_fd, int in_fd, address offset, int count) int
```

Transfer up to `count` bytes from `in_fd` to `out_fd` entirely in the kernel, without copying
data to user space. `offset` is a pointer to an `int` holding the file offset to read from in
`in_fd`; the kernel updates it after the call. Pass `0` for `offset` to use and advance the
current file position.

`out_fd` MUST be a socket or a regular file opened for writing. `in_fd` MUST support
`mmap`-style access (regular files, not sockets or pipes).

Returns bytes transferred, or a negative errno.

#### `splice`

```hopper
function splice(int fd_in, address off_in, int fd_out, address off_out, int len, int flags) int
```

Move up to `len` bytes between two file descriptors without copying to user space. At least
one of `fd_in` or `fd_out` MUST be a pipe. `off_in` and `off_out` are pointers to file
offsets; pass `0` to use and advance the current position.

`flags` is a bitmask; common values:

| Value | Meaning |
|-------|---------|
| `0` | Blocking operation |
| `1` | `SPLICE_F_MOVE` — hint to move pages rather than copy |
| `2` | `SPLICE_F_NONBLOCK` — do not block on pipe I/O |
| `4` | `SPLICE_F_MORE` — more data will follow; hint for TCP corking |

Returns bytes transferred, `0` on EOF, or a negative errno.

#### `tee`

```hopper
function tee(int fd_in, int fd_out, int len, int flags) int
```

Duplicate up to `len` bytes from the pipe `fd_in` to the pipe `fd_out` without consuming
them from `fd_in`. Both file descriptors MUST be pipes. Useful for implementing fan-out
without intermediate buffers.

Returns bytes duplicated or a negative errno.

#### `ioctl`

```hopper
function ioctl(int fd, int request, address arg) int
```

Device-control request. `request` is a device-specific integer constant. `arg` is a pointer
to a request-specific structure or a scalar cast to `address`. Returns `0` on success or a
negative errno.

Common uses include terminal settings (`TIOCGWINSZ`, `TCGETS`, `TCSETS`), socket options, and
network contract configuration (`SIOCGIFADDR`). The caller is responsible for knowing the
layout of the `arg` structure required by each `request`.

### Error Handling

All `IO` functions return a negative integer on error. The absolute value is the Linux errno
code. The `IO` contract does not throw exceptions and does not set a global `errno` variable
— the return value IS the error code.

```hopper
int n = read(fd, buf, 4096)
if (n < 0) {
    // error: n is -errno
    write(StdFd.STDERR, cast<address>("read failed\n"), 12)
}
```

Common errno values (as negative integers):

| Code | Meaning |
|------|---------|
| `-1` (`-EPERM`) | Operation not permitted |
| `-2` (`-ENOENT`) | No such file or directory |
| `-4` (`-EINTR`) | Interrupted by signal — retry the call |
| `-9` (`-EBADF`) | Bad file descriptor |
| `-11` (`-EAGAIN`) | Resource temporarily unavailable (non-blocking) |
| `-12` (`-ENOMEM`) | Out of memory |
| `-13` (`-EACCES`) | Permission denied |
| `-14` (`-EFAULT`) | Bad address |
| `-22` (`-EINVAL`) | Invalid argument |
| `-32` (`-EPIPE`) | Broken pipe |

The `IO` contract does not define errno constants. Programs that need them MUST define
their own `enum` or `const` declarations.

### Relationship to Other Interfaces

`IO` covers generic I/O on already-open file descriptors. To open and manage file descriptors
see `FileSystem` (§13.5). For socket-specific send/recv semantics see `Socket` (§13.6).
For memory-mapped I/O see `Memory` (§13.9).
