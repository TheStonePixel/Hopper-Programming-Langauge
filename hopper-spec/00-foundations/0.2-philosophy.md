## Philosophy

Hopper is built on a single principle:

> Hardware is not an implementation detail. It is the subject matter.

Most languages treat the hardware as something to abstract away. The goal is to let the programmer think in terms of algorithms, data structures, and types — and let the compiler, runtime, or operating system handle the messy hardware realities underneath. This is a reasonable position for most software.

Systems software is different. When you are writing a device driver, the hardware IS the algorithm. When you are writing a memory allocator, the memory model IS the data structure. When you are writing a scheduler, the CPU IS the type system. Abstracting the hardware away does not simplify the problem — it obscures the problem and makes it harder to reason about correctly.

Hopper's response is to make hardware visible, named, and composable — not to expose it as raw machine instructions, but to expose it as a structured part of the language's module system.

**Hardware capabilities are modules.** A program does not write to a UART by emitting `mov` instructions. It imports an contract that describes what I/O looks like on a UART, and the build system resolves that contract to the concrete hardware implementation for the target device.

**The compiler is a translator, not a policy maker.** The compiler's job is to convert Hopper source to correct LLVM IR. It does not decide how memory is managed, how casting works, or how the standard library behaves. Those decisions are made in source code that the programmer can read, inspect, and modify.

**Explicit over implicit.** If something happens at runtime — an allocation, a cast, a system call — it appears in the source code. There are no hidden operations. There is no implicit coercion. What you write is what executes.

**Source is the specification.** The module system, the standard library, the ABI — all of these are specified in Hopper source code, not in compiler internals. A conforming implementation is one that correctly executes that source code on the target platform.
