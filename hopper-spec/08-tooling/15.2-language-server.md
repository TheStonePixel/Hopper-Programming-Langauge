## Language Server

### LSP Status

**Not yet implemented.**

Hopper does not currently provide a Language Server Protocol (LSP) implementation. None of the
following LSP features are available:

- Hover documentation
- Go-to-definition
- Find references
- Code completion / IntelliSense
- Inline diagnostics / error squiggles
- Rename symbol
- Code actions
- Signature help
- Semantic tokens (beyond TextMate grammar coloring)
- Workspace symbol search

A full LSP implementation is a planned future deliverable. Until one exists, users SHOULD
rely on the VS Code extension for syntax-level editor support and on `hopper build` for
error reporting.

---

### VS Code Extension

The Hopper VS Code extension (`seed/extensions/hopper-vscode/`) provides syntax highlighting,
format on save, bracket matching, and comment toggling for `.hop` files. It does not provide
any LSP capabilities.

**Extension metadata:**

| Field | Value |
|-------|-------|
| Name | `hopper-language` |
| Display name | Hopper Language |
| Publisher | `hopper` |
| Version | 1.3.0 |
| Minimum VS Code version | 1.70.0 |
| Activation event | `onLanguage:hopper` |
| File extension registered | `.hop` |
| Language ID | `hopper` |
| Language aliases | `Hopper`, `hop` |

#### Syntax Highlighting

The extension registers a TextMate grammar at
`seed/extensions/hopper-vscode/syntaxes/hopper.tmLanguage.json` with the scope name
`source.hopper`. The grammar is activated for any file with the `.hop` extension.

A bundled dark theme is provided at `seed/extensions/hopper-vscode/themes/hopper-dark.json`.
In addition to the theme, the extension declares default `editor.tokenColorCustomizations`
rules in `package.json` that apply Hopper-specific coloring conventions regardless of the
active theme:

| Scope | Color |
|-------|-------|
| `keyword.memory.hopper` | `#E06C75` (bold) — `allocate`, `deallocate` |
| `keyword.class.hopper`, `entity.name.type.class.hopper` | `#E5A550` |
| `variable.language.self.hopper` | `#E5A550` (italic) |
| `keyword.interface.hopper`, `entity.name.type.interface.hopper` | `#56B6C2` |
| `keyword.function.hopper` | `#61AFEF` |
| `entity.name.function.hopper` | `#E5C07B` |
| `storage.type.hopper` | `#56B6C2` |
| `string.quoted.double.hopper` | `#98C379` |
| `comment.line.double-slash.hopper` | `#5C6370` (italic) |
| `keyword.control.hopper` | `#4FC1FF` |
| `keyword.operator.logical.hopper` | `#C678DD` |

#### Bracket Matching and Comment Toggling

Language configuration is declared in
`seed/extensions/hopper-vscode/language-configuration.json`.

**Brackets:** `{ }`, `[ ]`, `( )`. All three pairs participate in bracket matching and
indentation hinting.

**Auto-closing pairs:** `{ }`, `[ ]`, `( )`, `" "`, `' '`.

**Line comment:** `//`. The Toggle Line Comment command (`Ctrl+/` / `Cmd+/`) prefixes or
removes `//` on the selected lines.

Block comments are not defined; Hopper has no block comment syntax.

#### Format on Save

The extension registers a `DocumentFormattingEditProvider` for the `hopper` language that
delegates to `hopper fmt`. Format on save is enabled by default for `.hop` files:

```json
"[hopper]": {
    "editor.defaultFormatter": "hopper.hopper-language",
    "editor.formatOnSave": true
}
```

When VS Code invokes the formatter, the extension:

1. Writes the document's current text to a temporary file in the system temp directory
   (named `_hopper_fmt_<timestamp>.hop`).
2. Calls `hopper fmt "<tmp>" --write` via `execSync` with a 10-second timeout.
3. Reads the reformatted content back from the temporary file.
4. Returns a single `TextEdit` that replaces the entire document range with the formatted
   content. If the formatted content is identical to the original, an empty edit array is
   returned (no-op).
5. Deletes the temporary file in a `finally` block regardless of success or failure.

If the formatter exits with a non-zero status, the extension shows a warning notification
(first line of stderr) and returns an empty edit array, leaving the document unchanged.

#### `hopper.executablePath` Setting

The extension contributes a single user-configurable setting:

| Setting | Type | Default | Description |
|---------|------|---------|-------------|
| `hopper.executablePath` | `string` | `""` | Path to the `hopper` binary |

The extension resolves the `hopper` binary using the following priority:

1. The value of `hopper.executablePath` if non-empty.
2. A dev monorepo path at `<extension-dir>/../../build_system/hopper` — resolves to
   `seed/build_system/hopper` when the extension runs from `seed/extensions/hopper-vscode/`.
3. The string `"hopper"` — relying on `PATH`.

#### Installation

The VS Code extension MUST be installed by running:

```
node seed/build_system/tools/install-ext.js
```

The installer creates a symlink from `~/.vscode/extensions/hopper.hopper-language-<version>`
(or `~/.vscode-server/extensions/` for remote sessions) to the extension directory. VS Code
must be reloaded after installation.

There is no VSIX package; the extension is not published to the VS Code Marketplace.
