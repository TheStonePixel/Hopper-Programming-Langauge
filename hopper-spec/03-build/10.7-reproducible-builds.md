## Reproducible Builds

### Determinism of the Compilation Pipeline

Given identical inputs and the same compiler version, the Hopper build system produces
deterministic output at each stage:

- **Parsing and AST construction** — the ANTLR4 grammar is deterministic; identical source
  text produces an identical parse tree and AST.
- **Code generation** — `genModule()` produces identical LLVM IR text for identical input
  ASTs; no random values or timestamps are embedded in the IR.
- **IR deduplication** — `dedupeDeclarations()` is a deterministic filter; the first
  occurrence of each unique `declare` line is kept, in the order they appear.
- **Clang compilation** — given the same IR, the same clang version, and the same flags,
  clang produces a deterministic object file.

### Absolute Paths in Debug Info

The `dev` and `test` build profiles pass `-g` to clang, which embeds DWARF debug information
in the object file. DWARF debug info includes absolute filesystem paths to the source files.
These paths vary by machine (the username, home directory, or repository clone location may
differ).

As a result, debug builds are NOT byte-for-byte reproducible across machines, even when all
source content is identical.

Production builds (`--production`) do not pass `-g`, so they do not embed source paths in
debug info. Production object files and executables are byte-for-byte reproducible across
machines, subject to the clang version being identical.

### Build Metadata Timestamp

The bare-metal build writes a `<name>.meta.json` file containing a `builtAt` field with the
ISO 8601 wall-clock time of the build (see §10.6). This field differs between builds performed
at different times, even if all source inputs are unchanged.

**Not yet implemented.** The `builtAt` field cannot be suppressed. To achieve fully
reproducible bare-metal build metadata, a future implementation SHOULD omit `builtAt` or
replace it with a content hash or VCS commit hash derived from the source inputs.

### Full Reproducible Build Guarantees

**Not yet implemented.** The build system does not currently provide any formal guarantee of
bit-for-bit output reproducibility. A full reproducible build implementation would require:

- Eliminating absolute paths from all build outputs (use `-fdebug-prefix-map` for DWARF).
- Replacing the `builtAt` timestamp in `.meta.json` with a deterministic value (e.g. a
  source hash or a `SOURCE_DATE_EPOCH` environment variable).
- Validating that the installed clang and lld versions are pinned and verified.

None of these measures are implemented today.
